# 贡献指南

欢迎贡献，但必须保持工作流通用、安全、可维护。

## 原则

- 保持对多种 AI 编程工具可用。
- 用小而清晰的规则替代巨大的 prompt 块。
- 线上站点安全优先：备份、保护数据、回滚说明、QA 证据。
- 除非明确说明新架构，否则保持固定技术栈。
- 不要加入密钥、凭据、私有站点数据或客户内容。

## 提交 PR 前

- 本地运行密钥扫描。
- 确认没有规则绕过上线闸门。
- 确认没有 snippet 鼓励全局 DOM 循环、自动提交购物车、不安全权限或硬编码密钥。
- reference 文件应能从 `SKILL.md` 直接发现。
- 新增行为需要更新文档。

## PR 检查项

- [ ] 没有密钥或私有数据。
- [ ] 没有客户专属凭据、URL、订单或客户信息。
- [ ] 尽量兼容 Codex、Claude、Trae、Cursor、OpenHands、Aider。
- [ ] 新 snippet 规则包含生命周期、作用范围、回滚方式和验证方式。
- [ ] 按钮/交互仍要求真实点击/输入测试。
- [ ] CSV/媒体导入完整性仍为硬要求。
- [ ] 上线模式仍必须等页面、产品、文章、SEO 和完整 QA 通过。
