# Roadmap

This roadmap is directional. Safety, maintainability, and verification quality matter more than speed.

## Near Term

- Expand helper script coverage for official WooCommerce CSV variants.
- Add more sample `site_config` files for non-English markets.
- Add richer Rank Math mapping examples for products, posts, pages, and taxonomies.
- Improve documentation for existing-site SEO optimization without layout rebuilds.
- Add more reusable QA report templates.

## Mid Term

- Add optional browser-automation recipes for common WordPress admin flows.
- Add more compatibility notes for Claude, Cursor, OpenHands, and Aider.
- Add fixtures for cart quantity, product quantity, checkout boundary, and mobile overflow QA.
- Add release automation once the workflow stabilizes.

## Long Term

- Build a test corpus of anonymized WooCommerce CSV structures.
- Add adapters for multiple AI-agent skill/package formats.
- Add stronger schema validation for `site_config` and SEO mapping files.
- Maintain a library of market-specific layout and content rules.

## Non-Goals

- No hardcoded live-site credentials.
- No custom theme framework unless explicitly requested.
- No guarantee of search ranking.
- No unsafe one-click production launch without QA evidence.
