# 中断续做协议

AI 编程工具可能因为服务器错误、上下文限制、浏览器崩溃、REST 超时、网络故障或用户暂停而中断。本项目必须能安全续做。

## 核心规则

从“最后一个已验证检查点”继续，而不是从“最后一次尝试操作”继续。

没有只读恢复检查证明必要时，禁止重复执行破坏性工作。

## 必需续做台账

每次运行都应保存或重建台账：

- 网站任务状态和 skill 任务状态。
- 当前阶段和最后一个已验证 phase gate。
- 已创建/更新的 WordPress ID：页面、文章、产品、媒体、菜单、snippets、模板。
- active snippets、生命周期、回滚说明、影响页面。
- WooCommerce 页面绑定和核心设置。
- 产品 CSV 源行数、导入/更新数量、失败行、图库/正文图片验证状态。
- 文章 ID、状态、排期、关键词、特色图、内链。
- 已通过和待完成 QA。
- 临时凭据/session 是否创建、是否撤销。
- 下一步安全操作。

## 续做步骤

1. 读取最新台账和进度/最终说明。
2. 任何写操作前先做只读 live state 检查：
   - 页面
   - WooCommerce 绑定
   - snippets
   - 产品/媒体数量
   - 文章状态
   - 菜单
   - Rank Math sitemap
   - 购物车/产品页/结账可用性
3. 对比 live state 和台账。
4. 撤销或轮换中断前创建的临时凭据。
5. 从最小安全未完成步骤继续。
6. 下一次高风险操作前更新台账。

## 禁止自动重复

- 旧站清理
- 删除产品
- 产品 CSV 导入
- 媒体导入
- 文章发布或排期
- 删除页面
- 替换 snippets
- 上线/索引提交

## 如果台账缺失

先通过只读 live state 检查创建最小恢复报告。只有识别出受保护数据和上线 blocker 后，才能进入 repair/recovery 模式继续。
