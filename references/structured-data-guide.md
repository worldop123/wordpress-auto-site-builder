# Structured Data and Schema Markup Guide

## What Is Structured Data
Structured data is a standardized format for providing information about a page and classifying the page content. Google uses structured data to understand page content and enable rich results (enhanced search appearances) that can significantly increase click-through rates.

Case studies: Rotten Tomatoes +25% CTR, Food Network +35% traffic, Nestle +82% CTR, Rakuten 1.5x time on page.

Read this reference when configuring Rank Math Schema Templates or adding JSON-LD to page HTML. Also read `rank-math-seo-guide.md` for Rank Math schema module configuration.

## Supported Formats

### JSON-LD (RECOMMENDED)
- JavaScript notation in `<script type="application/ld+json">` tags
- Easiest to implement and maintain
- Can be placed in `<head>` or `<body>`
- Google can read dynamically injected JSON-LD
- Recommended by Google for all structured data

### Microdata
- HTML specification for nesting structured data in HTML content
- Uses HTML tag attributes (`itemscope`, `itemtype`, `itemprop`)
- More difficult to maintain than JSON-LD

### RDFa
- HTML5 extension for linked data
- Uses HTML tag attributes
- Rarely used in modern WordPress implementations

## Schema.org vs Google Documentation
- Schema.org is the vocabulary source
- Google Search Central documentation is the FINAL authority for what Google supports
- Many schema.org properties are NOT used by Google
- Only implement properties documented by Google

## Ecommerce Schema Types (Critical for WooCommerce)

### Product Schema
Required properties:
- `name`: Product name
- `image`: Product image URL
- `description`: Product description
- `brand`: Brand name
- `offers`: Offer object with:
  - `price`: Price
  - `priceCurrency`: Currency code (USD, EUR, etc.)
  - `availability`: InStock, OutOfStock, PreOrder
  - `url`: Product URL
  - `priceValidUntil`: Price validity date
  - `itemCondition`: NewCondition, UsedCondition, etc.

Recommended properties:
- `sku`: Stock keeping unit
- `gtin13` / `gtin8` / `gtin12`: Global trade item number
- `mpn`: Manufacturer part number
- `review`: Review objects
- `aggregateRating`: Average rating with count
- `color`, `size`, `material`: Product attributes

Example JSON-LD:
```json
{
  "@context": "https://schema.org/",
  "@type": "Product",
  "name": "Example Product Name",
  "image": ["https://example.com/product.jpg"],
  "description": "Product description here",
  "brand": {"@type": "Brand", "name": "BrandName"},
  "sku": "SKU123",
  "offers": {
    "@type": "Offer",
    "url": "https://example.com/product",
    "priceCurrency": "USD",
    "price": "29.99",
    "availability": "https://schema.org/InStock",
    "itemCondition": "https://schema.org/NewCondition"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.5",
    "reviewCount": "128"
  }
}
```

### Review Schema
- For individual product reviews
- `reviewRating`: ratingValue, bestRating, worstRating
- `author`: reviewer name
- `datePublished`: review date
- `reviewBody`: review text

### AggregateRating Schema
- `ratingValue`: average rating
- `reviewCount` or `ratingCount`: number of reviews
- Must be backed by real reviews on the page

### Offer Schema
- `price`: numeric price (no currency symbol)
- `priceCurrency`: ISO 4217 currency code
- `availability`: schema.org URL
- `priceValidUntil`: date
- `shippingDetails`: shipping info (recommended for Merchant Center)

## Content Schema Types

### Article Schema
- `headline`: Article title
- `image`: Featured image
- `datePublished` and `dateModified`
- `author`: Author with name
- `publisher`: Organization with logo

### FAQPage Schema
- Array of `Question` objects
- Each Question has `acceptedAnswer` with `text`
- Enables FAQ rich results in search
- Very effective for CTR improvement

Example:
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "What is the shipping time?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "We ship within 1-2 business days. Delivery takes 3-5 days."
    }
  }]
}
```

### HowTo Schema
- Step-by-step instructions
- `step` array with `HowToStep` objects
- Each step has `name`, `text`, and optional `image`
- Enables HowTo rich results

### BreadcrumbList Schema
- `itemListElement` array with `ListItem` objects
- Each item has `position`, `name`, `item` (URL)
- Critical for breadcrumb rich results
- Often auto-generated by Rank Math breadcrumbs

### Organization Schema
- `name`, `url`, `logo`, `sameAs` (social profiles)
- `contactPoint`: phone, email, contact type
- `address`: PostalAddress object
- Used for Knowledge Graph

### LocalBusiness Schema
- Extends Organization with local business data
- `openingHoursSpecification`: business hours
- `geo`: latitude/longitude
- `priceRange`: price level indicator
- Critical for Local SEO

### Person Schema
- `name`, `jobTitle`, `image`
- `worksFor`: organization
- `sameAs`: social profiles
- Used for author Knowledge Graph

### WebSite Schema
- `name`, `url`
- `potentialAction`: SearchAction for site search
- Enables Sitelinks Search Box

### VideoObject Schema
- `name`, `description`, `thumbnailUrl`
- `uploadDate`, `contentUrl` or `embedUrl`
- `duration`: ISO 8601 duration format
- Enables video rich results

## Rank Math Schema Implementation

### Auto-Generated Schema (Free)
Rank Math automatically generates base schema for:
- Posts: Article schema
- Pages: WebPage schema
- Products: Product schema (with WooCommerce module)
- Categories: CollectionPage schema

### Schema Templates (Rank Math)
1. Go to Rank Math → Schema Templates
2. Create custom schema per post type
3. Map properties to custom fields or post data
4. Set default schema for each post type

### Custom Schema Builder (Pro)
- Drag-and-drop property builder
- Create custom schema types
- Stack multiple schemas on one page
- Conditional schema display rules

### Manual JSON-LD in Elementor HTML
For pages using Elementor HTML blocks, add JSON-LD directly:
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [...]
}
</script>
```

## Schema Validation

### Google Rich Results Test
- URL: https://search.google.com/test/rich-results
- Test by URL or paste code directly
- Shows detected schema types and any errors/warnings
- Preview how the page appears in search results

### Schema Markup Validator
- URL: https://validator.schema.org/
- Validates against schema.org vocabulary (broader than Google)
- Useful for checking non-Google schema properties

### Search Console Monitoring
- Rich Results report: shows pages with valid/invalid schema
- Monitor for drops in rich results coverage
- Fix errors promptly to maintain rich result eligibility

## Schema Guidelines and Policies

### Must Follow
- Data must accurately describe the page content
- All required properties must be present
- Don't create empty pages just for structured data
- Don't add data about content not visible to users
- Use the most specific schema type available

### Common Violations
- Missing required properties
- Data doesn't match visible content
- Using schema for content not on the page
- Manipulating review ratings or counts
- Marking up hidden or non-visible content
- Using Product schema on non-product pages

### Review and Rating Guidelines
- Ratings must come from genuine user reviews on your site
- Don't fabricate reviews or ratings
- Don't aggregate ratings from other sites without permission
- Review count must match actual reviews on the page
- Snippet ratings must be backed by visible reviews

## Recommended Schema Implementation for WordPress Sites

### Global Schema (all pages)
- Organization or LocalBusiness (site-wide)
- WebSite with SearchAction
- BreadcrumbList (auto via Rank Math)

### Product Pages
- Product schema with offers, brand, SKU
- AggregateRating (if reviews exist)
- Review schema (for individual reviews)
- BreadcrumbList

### Blog Posts
- Article schema with author, date, publisher
- FAQPage (if article has FAQ section)
- BreadcrumbList

### FAQ Page
- FAQPage schema with all questions and answers

### About Page
- Organization or Person schema

### Contact Page
- LocalBusiness or Organization with contactPoint

### Homepage
- Organization schema
- WebSite schema with SearchAction

## Schema and WooCommerce Product CSV

When rewriting product CSV for SEO:
- Product schema is auto-generated by Rank Math WooCommerce module

- Add GTIN/MPN/Brand fields in Rank Math Pro for richer product schema
- Test sample products with Rich Results Test after import

## Troubleshooting Schema Issues

### Schema Not Detected
- Verify JSON-LD is valid JSON (no syntax errors)
- Check `@type` is a valid schema.org type
- Ensure all required properties are present
- Verify Rank Math schema module is enabled
- Clear cache and retest

### Rich Results Not Appearing
- Google may take days to weeks to show rich results
- Not all valid schema results in rich results
- Check Search Console Rich Results report for errors
- Ensure content quality meets Google guidelines
- Verify there are no manual actions

### Duplicate Schema
- Don't output same schema type multiple times on one page
- If using both Rank Math auto-schema and manual JSON-LD, disable auto-schema for that page
- Check for conflicting SEO plugins outputting duplicate schema
